# DreamCo 1.0 Consolidation Scorecard

DreamCo's next phase is consolidation and proof. The platform should stop expanding into unrelated new systems until the core repository, registry, CI, Buddy routing, dashboard, and five sellable products are demonstrable.

## Current Problem

The repository has many generated systems, bot profiles, dashboards, reports, and workflows. The risk is that generated files and profile counts can look like working products. DreamCo 1.0 should prove what works, classify what does not, and route every gap into a clear rebuild or close decision.

## Priority Order

1. Stabilize the repository.
2. Establish the truth about every bot.
3. Fix public identity and documentation.
4. Make Buddy the real control system.
5. Make the dashboard operational.
6. Build trustworthy autonomy and learning.

## First Ten Updates

1. Correct registry and README counts.
2. Classify all profiles by implementation status.
3. Triage open pull requests.
4. Repair the required CI pipeline.
5. Complete the Command Center consolidation decision.
6. Make Buddy use the master registry for real routing.
7. Enforce approval, risk, spending, and autonomy.
8. Launch the Bot Fleet and Failure Center.
9. Make five bot products work end to end.
10. Publish a DreamCo 1.0 readiness scorecard.

## Production-Ready Gate

DreamCo should not be labeled production-ready until installation, login, Buddy routing, one complete workflow, approvals, data ownership, billing, monitoring, rollback, and five customer products pass documented acceptance tests.
