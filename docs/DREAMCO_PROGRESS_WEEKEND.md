# DreamCo Progress Weekend

The goal is to make the current DreamCo system reliable, organized, and demonstrable before adding more new projects.

## Order of Work

1. Verify the GitHub repository: correct remote, clean branch status, pull/push path, and visible Actions failures.
2. Complete the bot inventory: purpose, inputs, outputs, revenue opportunity, API notes, status, and test path.
3. Finish Buddy as the central dashboard: divisions, bot launch packets, errors, docs, rebuilds, and debug routing.
4. Test the Mac and GitHub workflow: clone, pull, push, and confirm GitHub receives a safe change.
5. Pick five sellable bots: define the customer, signup path, usage flow, pricing, and demo script.

## Sunday Night Definition of Done

- One verified GitHub repository.
- Complete bot inventory.
- Buddy central dashboard.
- Top five production candidates.
- Biggest bugs and missing features list.

## Rule

Do not expand into unrelated new projects until the current repository, dashboard, inventory, top five bots, and bug list are demonstrable.
