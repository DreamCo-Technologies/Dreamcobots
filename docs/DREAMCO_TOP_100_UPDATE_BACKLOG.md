# DreamCo Top 100 Update Backlog

This backlog converts the Top 100 update comparison into six build categories. The goal is to turn DreamCo into one reliable operating platform before expanding into more unrelated systems.

## Update Groups

| Group | Count | First Move |
| --- | ---: | --- |
| Repository stability | 20 | Make one required CI workflow and classify open pull requests. |
| Bot truth registry | 20 | Generate the public catalog only from the master registry. |
| Public docs and identity | 15 | Keep README claims tied to measurable evidence. |
| Buddy control plane | 15 | Replace first-match routing with scored registry routing. |
| Operational dashboard | 15 | Launch Bot Fleet and Failure Center as official dashboard surfaces. |
| Trustworthy autonomy | 15 | Enforce Guided, Semi-Automatic, and Full Automatic as code policies. |

## Rule

Treat the backlog as proof work, not idea expansion. Generated files, profiles, and dashboards should not be considered working products until they have implementation status, tests, owners, evidence, and dashboard visibility.

## First Products

Start with five end-to-end bot products:

- Lead Generation.
- Outreach.
- Follow-Up.
- Code and Repository Health.
- Small-Business Operations.
