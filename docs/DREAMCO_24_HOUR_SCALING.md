# DreamCo 24-Hour Scaling

DreamCo 24-hour scaling keeps the platform improving through safe, supervised cycles. It is not uncontrolled autonomy. The system researches, drafts, builds, tests, packages, and reports continuously while risky live actions wait for owner approval.

## Daily Cycle

- 00:00-03:00: market research, competitor notes, customer problem discovery, source logging.
- 03:00-07:00: app concepts, MVP scopes, sandbox fixtures, pull-request-safe build tasks.
- 07:00-11:00: syntax checks, smoke tests, API sandbox tests, workflow generators, report refresh.
- 11:00-15:00: app-store listings, prospectus pages, demo scripts, mockups, pricing drafts.
- 15:00-19:00: marketing drafts, customer discovery plans, SEO briefs, launch experiments.
- 19:00-24:00: scorecards, failure summaries, approval queue, and next-best build tasks.

## Scaling Targets

- Minimum replicas: 2.
- Maximum replicas: 20.
- CPU target: 65%.
- Memory target: 75%.
- Queue depth target: 100.
- Cooldown: 300 seconds.
- Self-healing: enabled.

## Always Blocked Without Approval

- Customer outreach.
- Ad spend.
- Money movement.
- Trading or investment actions.
- Public deployment.
- App-store publishing.
- Payment link publishing.
- Credential changes.
- Third-party account changes.
- Legal, medical, financial, or income claims.
- Destructive repository actions.

## Commands

npm run report:24-hour-scaling
